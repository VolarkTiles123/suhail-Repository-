## Content Cluster Architecture

### Pillar → Cluster Model

```
PILLAR PAGE (1 per topic)
Main page covering topic broadly
2000-3000 words
Links to all related clusters
├── CLUSTER 1 (narrow topic)
│   └── Supports pillar, builds topical authority
│
├── CLUSTER 2
│   └── Addresses specific intent/audience
│
├── CLUSTER 3
│   └── Provides deep dive / case study
│
└── CLUSTER 4
    └── Answers related questions
```

### Example: Porcelain Tiles

**Pillar Page:** "Porcelain Tiles - Complete Guide"
- URL: /porcelain-tiles/
- Length: 2500 words
- Intent: Informational + Commercial
- Links to all clusters internally

**Cluster Articles:**
1. "Types of Porcelain Tiles" (1000 words)
2. "Porcelain vs Ceramic Tiles" (1200 words)
3. "Porcelain Tiles for Bathrooms" (1000 words)
4. "Porcelain Tile Installation Guide" (1500 words)
5. "Porcelain Tile Maintenance" (800 words)
6. "Best Porcelain Tile Sizes" (1000 words)

Goal: Build authority + sales

Goal: Build topical authority

- "Porcelain Tile Trends 2024"
- "Design Styles with Ceramic Tiles"

Format: Blog, educational, trend report

## Internal Linking Structure

### Pillar ← → Clusters

**Pillar page links:**
- To all cluster articles (2-4 links)
- Uses descriptive anchor text
- Early in content (within first 500 words)
- Clear navigation structure

**Cluster pages link:**
- Back to pillar (1-2 links)
- To sibling clusters (1-2 links)
- To related clusters in other topics (1-2 links)

**Example:**
```
Porcelain Tiles (Pillar) links to:
├─ "Types of Porcelain" cluster
├─ "Porcelain vs Ceramic" cluster
├─ "Bathroom Porcelain" cluster
└─ "Installation Guide" cluster

"Types of Porcelain" links back to:
├─ "Porcelain Tiles" pillar
├─ "Porcelain vs Ceramic" (sibling)
└─ "Ceramic Tiles" (related topic pillar)
```
